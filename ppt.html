1. 项目概述与定位
PRODUCT OVERVIEW
03 / 16
🎯
定位：面向老人/行动不便人群的智能陪诊服务 SaaS 平台，连接患者、陪诊师、医院、运营管理员四方，实现"需求提交 → AI 解析 → 派单 → 医院对接 → 服务跟踪 → 评价反馈"的全链路闭环。

1解决什么问题

  • 老人子女不在身边，看病无人陪同
  • 挂号、排队、取药流程复杂，老人搞不定
  • 陪诊师资源分散，缺乏统一调度
  • 服务质量难追踪，出问题无据可查

2核心价值

  • 患者端：AI 说话填单 + 进度透明 + 紧急联系
  • 陪诊师：统一接单 + 智能匹配（专长+区域）
  • 医院：申请对接 + 后台审批 + 累计订单
  • 管理员：SaaS 后台 + 风险预警 + 审计日志

2. 技术底座选型
TECH STACK
04 / 16

前端基础

  • HTML5 + CSS3 + 原生 JavaScript（无任何框架）
  • ES6+ 语法：模板字符串、async/await、解构、箭头函数
  • CSS Variables（设计 token 系统，对齐科研2 SaaS）
  • 无构建工具，改完即上线，无 webpack/vite

样式系统

  • Noto Sans SC（正文）+ Noto Serif SC（标题）
  • 系统字体 fallback：PingFang SC / 微软雅黑
  • 设计 token：主色 #3b6cb5、圆角 8px/12px
  • SVG 线条图标库（lucide-react 风格，约 25 个）

AI 集成

  • DeepSeek Chat API（OpenAI 兼容协议）
  • REST 接口：POST /v1/chat/completions
  • 模型：deepseek-chat，temperature=0.3
  • 本地 fallback：parseSpokenNeed() 口语化解析
  • API Key 存浏览器 localStorage

部署架构

  • GitHub Pages 静态托管（零成本）
  • 仓库：16zyc/huwuyou-platform
  • git push → 自动构建 → 全球 CDN
  • 无后端服务器、无数据库（数据在内存）
💡
选型理由：本项目为演示/POC 阶段，纯前端架构保证"零运维成本 + 极速迭代"。后续若投入生产，需在后端补 FastAPI + PostgreSQL + Redis，前端可平滑迁移到 Vue3/React（核心逻辑都在 JS 模块对象里，可移植性好）。
技术栈极简，一个会 JavaScript 的工程师即可接手

数据生命周期：所有数据仅存内存（页面刷新即丢失），适合演示。生产化要点：将每个 Pool 改为 FastAPI 后端 + PostgreSQL 表，前端 fetch 调用即可，业务逻辑零改动。
// data.js 中 NeedPool 的定义（典型 Pool 模式）
const NeedPool = {
  list: [], seq: 1,
  add(need) {
    const id = 'N' + String(Date.now().slice(-6)) + this.seq++;
    const n = Object.assign({ id, status: '待处理', createTime: now, ... }, need);
    this.list.unshift(n); return n;
  },
  update(id, patch) { const n = this.getById(id); if (n) Object.assign(n, patch); }
};
所有数据池采用统一的 Pool 模式，扩展新池只需 5 行代码

10. AI 集成方案（DeepSeek）
AI INTEGRATION
12 / 16

🎯 三层降级策略

  • 第一层：快捷指令匹配（如"今天"+"安排"）→ 直接返回 MockData.todaySchedule
  • 第二层：DeepSeek API 真实调用（带 system prompt）
  • 第三层：API 失败时 fallback parseSpokenNeed() 本地口语化解析

🤖 System Prompt 设计

  • 注入当前患者完整档案（病史/过敏/用药/联系人）
  • 列出可选医院/科室/服务类型+价格
  • 要求返回纯 JSON：{"action":"create_need",...}
  • 信息不全时用自然语言追问
  • 禁止杜撰（防幻觉）

📡 API 调用核心代码（app.js）

const resp = await fetch(AIConfig.baseUrl + '/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + AIConfig.apiKey
  },
  body: JSON.stringify({
    model: 'deepseek-chat',
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: q }
    ],
    temperature: 0.3, max_tokens: 500
  })
});
const data = await resp.json();
const content = data.choices?.[0]?.message?.content?.trim();
// 兼容 markdown 代码块 ```json ... ```
const jsonStr = this.extractJSON(content); // 提取 JSON
if (jsonStr) {
  const parsed = JSON.parse(jsonStr);
  if (parsed.action === 'create_need') return this.createNeedFromAI(parsed, q);
}

🛡️ 本地 fallback 解析器

parseSpokenNeed(q) 支持：

  • 医院简称：协和/同仁/301/三零一
  • 科室口语：心/心脏/心血管→心内科，脑/脑梗→神经内科，腿/腰→骨科
  • 日期：下周X、后天、明天、周X
  • 服务类型：全程/半程/代办/复诊

🔑 API Key 管理

  • 存储：localStorage['huwuyou_ai_key']
  • 默认 Key 在 data.js 中（演示用）
  • 后台「系统设置」可改 Key
  • 生产环境务必用后端代理，不能暴露在前端
AI 返回的 [文字](#action) 链接会被 renderAiMessage() 转成可点击按钮
